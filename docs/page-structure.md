# ページの基本構成

このプロジェクトでは、各ページの見た目や情報の出し方を揃えるために、まず共通の骨組みを決める。
ここでは、すべてのツールページに共通する基本構成をまとめる。

## 基本方針

- 1ページ1機能を基本にする。
- 画面の上から下へ、入力 -> 操作 -> 結果 の流れが自然に読めるようにする。
- 迷ったら、同じ役割の要素は同じ見た目と順番にそろえる。
- 例外が必要な場合でも、ページ内で理由が分かるようにする。

## 標準レイアウト

1. `main` でページ全体を包む。
2. 左右の余白と最大幅を決める。
3. 先頭にトップへ戻る導線を置く。
4. ページタイトルを置く。
5. 必要なら使い方やサンプルのボタンを置く。
6. 入力エリアを置く。
7. 実行・解析・変換などの操作ボタンを置く。
8. 結果や状態表示を置く。
9. 補足情報や注意事項を最後にまとめる。

## 推奨セクション順

### 1. ナビゲーション

- 最初に「トップに戻る」を置く。
- 「トップに戻る」は `count` ページ方式を標準とする。
- 可能ならページ冒頭で戻り先を固定する。
- 迷うページでは、見出しより前に補助情報を置きすぎない。

```tsx
<div className="mb-4">
	<Link href="/" className="text-blue-500 hover:text-blue-700 text-sm">
		← トップに戻る
	</Link>
</div>
```

### 2. タイトル

- ページの役割が一目で分かる名前にする。
- 表示名はホーム画面のリンク名と揃える。

### 3. 補助操作

- 「使い方」「サンプル」などの補助操作は、タイトルの次の行に配置する。
- 「使い方」ボタンは count ページ位置を標準とし、タイトルの下に改行して置く。
- 「使い方」は押下でモーダルではなくページ内で展開する。
- 展開中はボタン文言を「閉じる」に切り替える。
- 「使い方」の中には閉じる以外のボタンは配置しない。
- 補助操作は主操作より目立たせすぎない。

```tsx
<h1 className="text-2xl font-bold mb-4">ページタイトル</h1>
<div className="flex items-center mb-4 gap-2">
	<button
		className={`px-4 py-2 rounded text-sm transition-colors ${
			showHelp
				? "bg-gray-200 text-gray-800 hover:bg-gray-300"
				: "bg-green-100 text-green-700 hover:bg-green-200"
		}`}
		onClick={() => setShowHelp((v) => !v)}
	>
		{showHelp ? "閉じる" : "使い方"}
	</button>
</div>
{showHelp && (
	<div className="mb-6 p-4 bg-blue-50 border border-blue-200 rounded text-sm leading-relaxed">
		{/* 使い方本文 */}
	</div>
)}
```

### 4. 入力

- 入力欄は、左から右、上から下のどちらかに統一する。
- 入力と出力を兼用する場合は、結果側に置く。
- 複数入力がある場合は、ラベルを短く、説明は補足に分ける。
- 文字列入力、数値入力、チェックボックス、選択肢の順番は意味のまとまりで並べる。

### 5. 実行

- 解析や変換を行う主要ボタンは、入力欄の近くに置く。
- 「解析、実行」「リセット」「エクスポート」「インポート」の順でボタンを配置する。 存在しないボタンは飛ばす。
- 主操作は 1 つを強く見せ、その他の操作は補助に回す。
- 自動更新するページでは実行ボタンは配置しない。
- 文言は原則「リセット」に統一する。
- 例外として、入力欄だけを消去するなど部分的な初期化を明示したい場合のみ「クリア」を使う。

### 5.1 ボタンの統一ルール

- ボタンの基本サイズは高さ 40px 相当を標準とする。
- 横並びボタンの余白は `gap-2` を標準とする。
- 角丸は `rounded` を標準とし、過度に丸くしない。
- 文字サイズは通常 `text-sm`、重要操作のみ `text-base` を許可する。

#### 色と役割

- 解析、実行: `bg-blue-600 text-white hover:bg-blue-700`
- サンプル、使い方: `bg-green-100 text-green-700 hover:bg-green-200`
- リセット: `bg-red-100 text-red-700 hover:bg-red-200`
- エクスポート: `bg-cyan-700 text-white hover:bg-cyan-800`
- インポート: `bg-slate-600 text-white hover:bg-slate-700`
- 閉じる、キャンセル: `bg-gray-200 text-gray-800 hover:bg-gray-300`

#### 推奨クラス

```tsx
// 標準ボタン
className="px-4 py-2 rounded text-sm"

// 盤面サイズ変更などの小型ボタン
className="h-[30px] px-3 rounded text-sm"
```

#### 実装上の注意

- 同じ役割のボタンはページをまたいでも同じ色を使う。
- 同じ行の中で、主操作だけを濃色にする。
- 無効状態が必要な場合は `disabled:opacity-50 disabled:cursor-not-allowed` を付与する。
- モーダル内の「使い方」では閉じる以外のボタンを置かない。

### 6. 結果

- 盤面のサイズ変更に関するボタンは盤面の真上に置く。
- 結果は入力欄より下に置く。
- 成功結果、未確定結果、エラーは表示場所を分ける。
- 長い結果は、要約と詳細を分ける。

### 7. 注意事項

- 仕様上の制約や前提条件は最後にまとめる。
- 入力例や具体例があるときは、注意事項の近くに置く。

## 共通コンポーネントの目安

- 戻るリンク
- ページタイトル
- 使い方の展開エリア
- サンプル入力ボタン
- リセットボタン
- 実行ボタン
- 結果表示エリア
- 注意事項

## 実装時の目安

- ページ固有の見た目は許可するが、骨組みは崩さない。
- 似た役割の UI は同じクラス名や同じ配置を使う。
- 新しいページを作るときは、この文書の順番を最初の雛形にする。

## 例外の扱い

- ホームページのような一覧画面は、この構成をそのまま使わず、導線中心のレイアウトにする。
- ゲーム盤面や大きなキャンバスが主役のページでは、入力と結果の位置を一部入れ替えてよい。
- ただし、タイトル、戻る導線、補助操作、注意事項の考え方は維持する。
